# Народные тропы

Веб-симуляция возникновения **Народных троп**: агенты движутся между точками интереса по карте проходимости, оставляют **Протоптанность**, а накопленная протоптанность постепенно делает повторяемые маршруты более удобными.

## Основные сущности

- **Карта проходимости** — чёрно-белое изображение `assets/initial-cost-map.png`, где каждая клетка показывает, насколько легко через неё пройти.
- **Точка интереса** — нормализованная координата на карте, откуда агент может появиться и куда может идти.
- **Агент** — движущаяся сущность, у которой есть позиция, направление, скорость, случайное отклонение и выбранная целевая точка интереса.
- **Протоптанность** — отдельная сетка накопленных следов агентов, которая повышает итоговую проходимость уже посещённых клеток.
- **Грубый маршрут** — поле накопленной стоимости прохода к конкретной точке интереса. Для каждой точки интереса строится отдельное поле.

## Системы координат и разрешение карт

Позиции агентов и точек интереса хранятся в нормализованных координатах:

```text
x: 0..1
y: 0..1
```

Для расчёта маршрутов canvas переводится в более грубую сетку:

```text
routeWidth  = round(canvas.width  / routeResolutionScale)
routeHeight = round(canvas.height / routeResolutionScale)
```

Чем больше `routeResolutionScale`, тем грубее сетка маршрутов и тем быстрее пересчёт.

## Исходная карта проходимости

Исходная карта читается из изображения `assets/initial-cost-map.png`. Для каждой клетки берётся средняя яркость RGB:

```text
basePassability = (R + G + B) / (255 * 3)
```

Диапазон:

```text
0..1
```

- `0` — чёрная непроходимая клетка.
- `1` — белая максимально проходимая клетка.
- Значения между `0` и `1` — частичная проходимость.

## Карта протоптанности

Протоптанность хранится в `trailGrid.values`. Изначально все значения равны `0`.

Когда агент проходит через клетку, он увеличивает значение протоптанности:

```text
newTrail = min(1, oldTrail + trailDepositRate * deltaTime)
```

Текущая константа:

```text
trailDepositRate = 0.018
```

Диапазон протоптанности:

```text
0..1
```

- `0` — клетка не протоптана.
- `1` — максимальная накопленная протоптанность.

## Итоговая проходимость с учётом протоптанности

Перед построением маршрутов исходная проходимость объединяется с протоптанностью:

```text
if basePassability == 0:
  finalPassability = 0
else:
  finalPassability = min(1, basePassability + trail * trailPassabilityBonus)
```

Текущая константа:

```text
trailPassabilityBonus = 0.45
```

Диапазон итоговой проходимости:

```text
0..1
```

Важные свойства:

- Протоптанность не делает чёрные непроходимые клетки проходимыми.
- Протоптанность повышает проходимость только там, где исходная проходимость больше `0`.
- Итоговая проходимость ограничена сверху значением `1`.

## Стоимость шага между клетками

При построении маршрута алгоритм оценивает переход из текущей клетки в соседнюю. Соседи рассматриваются в 8 направлениях: горизонтально, вертикально и диагонально.

Сначала берётся худшая проходимость на переходе:

```text
pathPassability = min(currentPassability, nextPassability)
```

Если `pathPassability` равна `0`, переход невозможен:

```text
stepCost = Infinity
```

Иначе стоимость шага:

```text
stepCost = stepLength / pathPassability^4
```

Где:

```text
stepLength = 1       для горизонтального или вертикального шага
stepLength = sqrt(2) для диагонального шага
```

Диапазон стоимости шага:

```text
1..Infinity
```

Низкая проходимость штрафуется очень сильно из-за степени `4`. Например для прямого шага:

| Проходимость | Стоимость |
|---:|---:|
| `1.0` | `1` |
| `0.5` | `16` |
| `0.25` | `256` |
| `0` | `Infinity` |

## Построение грубых маршрутов

Для каждой точки интереса строится отдельное поле `distance`.

В этом поле каждая клетка хранит минимальную накопленную стоимость прохода от этой клетки до выбранной точки интереса.

Алгоритм:

1. Создать массив `distance` размером `routeWidth * routeHeight`.
2. Заполнить его значениями `Infinity`.
3. Найти клетку целевой точки интереса.
4. Записать для неё `distance = 0`.
5. Добавить её в приоритетную очередь.
6. Пока очередь не пуста, доставать клетку с минимальной стоимостью и пробовать улучшить значения соседей.

Формула обновления соседа:

```text
nextDistance = currentDistance + stepCost
```

Если `nextDistance` меньше уже записанного значения соседа, значение обновляется.

Диапазон значений `distance`:

```text
0..Infinity
```

- `0` — сама целевая точка интереса.
- Конечное положительное число — достижимая клетка.
- `Infinity` — недостижимая клетка.

## Как агент выбирает направление движения

У агента нет собственного списка точек маршрута. Он использует общее поле `distance` для своей целевой точки интереса.

На каждом шаге агент:

1. Переводит свою позицию в координаты сетки маршрута.
2. Берёт текущее значение `distance` в своей позиции.
3. Просматривает локальное окно соседних клеток вокруг себя.
4. Оставляет только клетки, которые уменьшают стоимость пути к цели.
5. Суммирует направления к таким клеткам с весом.

Вес соседней клетки:

```text
improvement = currentDistance - neighborDistance
weight = improvement / (1 + localDistance)
```

Клетка участвует только если:

```text
neighborDistance is finite
improvement > 0
```

После суммирования итоговый вектор нормализуется до единичного направления.

Если поле маршрута недоступно или текущее значение не подходит, агент идёт напрямую в сторону целевой точки интереса.

## Билинейная интерполяция значений карты

Позиция агента непрерывная, а карты дискретные. Поэтому значения `passability` и `distance` берутся не только из одной клетки, а интерполируются между четырьмя соседними.

Для проходимости:

```text
top    = interpolate(topLeft, topRight, tx)
bottom = interpolate(bottomLeft, bottomRight, tx)
value  = interpolate(top, bottom, ty)
```

То есть фактически используются четыре значения:

```text
topLeft      = map[x0, y0]
topRight     = map[x1, y0]
bottomLeft   = map[x0, y1]
bottomRight  = map[x1, y1]
```

Итоговая формула:

```text
top    = topLeft + (topRight - topLeft) * tx
bottom = bottomLeft + (bottomRight - bottomLeft) * tx
value  = top + (bottom - top) * ty
```

Для `distance` применяется тот же принцип, но бесконечные значения игнорируются. Если все соседние значения бесконечны, результат тоже `Infinity`.

## Случайные отклонения агента

Агент не движется идеально по грубому маршруту. У него есть случайное отклонение направления.

Периодически выбирается новое случайное отклонение:

```text
angle = random(0..2π)
strength = random(0..wanderStrength)
targetWander = direction(angle) * strength
```

Затем текущее отклонение плавно приближается к целевому.

Перед применением отклонения агент проверяет проходимость впереди в кандидатном направлении:

```text
wanderScale = sampledForwardPassability^2
```

Диапазон:

```text
0..1
```

- `0` — отклонение полностью подавляется.
- `1` — отклонение применяется полностью.

Это помогает агентам меньше отклоняться в сторону непроходимых областей.

## Защита от шага в непроходимую область

Перед фактическим движением агент проверяет проходимость следующей позиции.

Если следующая позиция непроходима, агент:

1. Сбрасывает случайное отклонение.
2. Возвращает направление к грубому маршруту.
3. Продолжает движение уже без текущего отклонения.

## Жизненный цикл агентов

Агент появляется в случайной точке интереса и выбирает другую точку интереса как цель.

Когда агент достигает цели, он исчезает из симуляции.

Чтобы поддерживать примерно заданную численность, новые агенты появляются постепенно:

```text
missingAgents = targetAgentCount - currentAgentCount
spawnRate = max(minimumAgentSpawnRate, targetAgentCount / agentPopulationRecoveryTime)
spawnRemainder += spawnRate * deltaTime
spawnCount = min(missingAgents, floor(spawnRemainder))
```

Текущие константы:

```text
minimumAgentSpawnRate = 6
agentPopulationRecoveryTime = 8
```

Если пользователь уменьшает целевое число агентов, лишние агенты удаляются сразу.

## Визуализация

Каждый кадр рисуются:

1. Исходная карта проходимости.
2. Визуальный слой протоптанности.
3. Опциональная дебаг-карта грубого маршрута.
4. Точки интереса.
5. Агенты.

Протоптанность рисуется белым слоем с прозрачностью:

```text
alpha = min(0.45, trailValue)
```

Диапазон прозрачности:

```text
0..0.45
```
